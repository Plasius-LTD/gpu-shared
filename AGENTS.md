# AGENTS

## Scope
- `@plasius/gpu-shared` owns shared browser-safe demo/runtime helpers for the Plasius `gpu-*` family.

## Active Rules
- Keep this package browser-safe and independent from React-bound entrypoints.
- Prefer reusing existing `@plasius/gpu-*` package APIs over cloning package-local logic here.
- Shared showcase/runtime files belong here, not in `gpu-demo-viewer`.
- Do not add package-local analytics delivery; route any export needs through `@plasius/analytics`.
- Update `README.md`, `CHANGELOG.md`, ADRs, and TDRs whenever the shared runtime contract changes.

## Setup
- Use Node.js 24 (see `.nvmrc`) and npm.
- Install dependencies with `npm ci`.

## Common commands
- `npm test`
- `npm run lint`
- `npm run typecheck`
- `npm run build`

## Repo conventions
- Source lives in `src/`; tests in `tests/`.
- `dist/` is generated by `tsup` (do not edit by hand).
- Add tests for new or changed behavior.
- Architectural changes require ADR/TDR updates.

## Safety
- Never include secrets or real PII in code, tests, or demos.
- Follow `SECURITY.md` for vulnerability reporting.

## Package Identity Clarification
- Active rule: all package-identity, legal, contributor, and user-facing text in this repository must refer to `@plasius/gpu-shared`.
- Template provenance from other repositories is not package identity.

## Shared Runtime Governance
- `gpu-demo-viewer` may consume this package, but it must not be the source of truth for shared runtime code.
- When package demos need a common 3D validation surface, add or update it here first, then rewire consumers.
